# BOL Sorter & Creator

A custom Python tool built to streamline the process of sorting and packaging Bills of Lading (BOLs) for shipment.

## The Problem

BOLs are automatically generated by an existing upstream process that cannot be modified. This process outputs:

- A single large PDF containing all BOL pages
- Pages in **no particular order**
- No sorting by distribution center (DC)
- No accompanying summary documentation

Before shipment, these BOLs need to be:
1. Sorted by DC
2. Accompanied by a **Supplement file** and a **Master BOL file**, both listing key details pulled from each individual BOL page (BOL number, weight, quantity, PO #, load #, shipper info, recipient info, etc.)

Doing this manually was slow and error-prone, taking an estimated **20–30 minutes per BOL** to prepare.

## The Solution

A Python program that automates sorting and document generation using **preconfigured shipping profiles** and **pdfplumber** for PDF data extraction.

### Key Features

- **Shipping Profiles** — Users can create and save profiles per shipping account, storing shipper and recipient information so it doesn't need to be re-entered each time.
- **Minimal Manual Input** — For each shipment, the user only needs to:
  - Select the relevant shipping profile
  - Enter the master BOL number
  - Enter the pallet count
  - Enter the shipping date
  - Upload the large, unsorted BOL PDF
- **Automated Data Extraction** — Using `pdfplumber`, the program parses every page of the PDF to pull:
  - Load number
  - PO number
  - Individual BOL number
  - DC number
  - Weight
  - Quantity
- **Automatic Sorting** — Pages are reordered and grouped by DC number.
- **Document Generation** — The program produces:
  - A sorted BOL PDF
  - A **Master BOL** (.docx) summarizing all included BOLs, total weight, and total quantity
  - A **Supplement file** (.docx) with the detailed per-BOL breakdown

### Output

The final set of documents can be printed together as a complete, shipment-ready BOL package.

## Workflow Summary

1. User selects a saved shipping profile
2. User enters master BOL #, pallet count, and shipping date
3. User uploads the raw, unsorted BOL PDF
4. Program extracts per-page data via `pdfplumber`
5. Program sorts pages by DC
6. Program generates Master BOL and Supplement docx files
7. User prints the complete package

## Tech Stack

- **Python**
- **pdfplumber** — PDF text/data extraction
- **python-docx** — Master BOL and Supplement file generation

## Impact

- Eliminates manual sorting of large, unordered BOL PDFs
- Removes manual transcription of BOL details into summary documents
- Saves an estimated **20–30 minutes of manual work per BOL**
- Reduces the risk of transcription errors in shipment documentation
